CNStats 2.5 - Руководство пользователя

Просмотр статистики.

Для входа в систему необходимо ввести, указанный при установке, административный пароль. Если пароль введен не верно, то Вы будете возвращены к форме ввода пароля.

CNStats использует PHP сессии для авторизации. Если система Вас не пускает, то либо Вы неверно вводите пароль, либо функции работы с сессиями работают неверно.

Окно статистики разделено на три части

  • Левое меню - содержит список отчетов, разбитых на следующие блоки.
  • Основная часть - Содержимое отчета.
  • Правая колонка - содержит дополнительное меню, меню выбора даты и отчет о временных периодах и времени генерации отчета.

Дополнительное меню зависит от просматриваемого отчета, у некоторых отчетов его просто нет.

При первом заходе в систему, рекомендуем Вам зайти в раздел "Настройки" и обновить базу данных стран. О том, как это делается, подробно Вы сможете узнать, почитав описание отчета "Страны"

Вход в систему

Доступ к статистическим отчетам ограничен. Для входа в систему вы должны ввести логин и пароль. В качестве логина вы должны использовать E-Mail, который указыван в конфигурационном файле, а пароль вы указывали во время установки системы.

Галочка "Запомнить меня на этом компьютере" позволяет не вводить логин и пароль при каждом входе в систему.

Внимание: Устанавливайте эту галочку только на тех компьютерах, к которым никто кроме вас не имеет доступа.

Эти настройки вы можете легко поменять в файле config.php.

Регистрационный E-Mail: 

$STATS_CONF["cnsoftwarelogin"]="your@email.com";

Пароль: 

$STATS_CONF["adminpassword"]="ac4cb9578439ad456f4567b8da5f02";

или (менее безопасно): 

$STATS_CONF["adminpassword"]=md5("youpassword");

Кроме того, вы можете вообще отключить авторизацию. Это бывает нужно, если вы используете свою систему авторизации и CNStats является частью системы управления сайтом: 

$COUNTER["disablepassword"]="yes";

Предварительные фильтры

Предварительные фильтры позволяют уточнять существующие отчеты, ниже приведены примеры использования предварительных фильтров:

  • Из отчета "Ссылающиеся страницы" можно исключить поисковые системы
  • Из отчета "Ссылающиеся страницы" можно исключить страницы вашего сайта
  • Если ваш сайт имеет несколько доменов, то можно построить любой отчет для одного из них.
  • Можно построить любой отчет для конкретной страницы вашего сайта
  • Можно построить любой отчет для посетителей, использующих браузер "Опера"

Для использования предварительного фильтра необходимо выбрать его в нижней части страницы с отчетом.

Описания отчетов

Отчет "Посещаемость"

Отчет отображает изменение посещаемости Вашего ресурса за определенный период времени. При построении отчета по сводной таблице в зависимости от настроек периоды следующие:

СуммированиеПериод
По часам1 день
По дням44 дня
По неделям307 дней
По месяцам1308 дней

Если выбран фильтр,то отчет строится по данным лог-таблицы за период, определяемый переменной 

$COUNTER["savelog"]
которая находится в файле config.php. Максимальная величина - 30 дней.

Хиты - количество загрузок страниц сайта.

Хосты - количество уникальных IP-адресов, с которых запрашивались страницы сайта. Несколько компьютеров за одним неанонимным прокси-сервером и за разными анонимными прокси считаются за несколько хостов.

Посетители - количество уникальных посетителей сайта (точнее, количество браузеров). Идентификация происходит с помощью cookies. Правильный подсчет посетителей работает, только в том случае, если вы используете графический GIF счетчик (*).

С помощью иконок , и Вы можете изменять внешний вид графика, кроме того, Вы можете усреднять отображаемые на графике значения.

(*) Функцией include стоит пользоваться только для служебных страниц сайта.

Отчет "Популярные страницы"

Без имени хоста: 

/
/cnstats/
/cnstats/shots.xhtm

Помимо имени хоста, с помощью пунктов "Полный URL" и "Сокращенный URL" можно отбрасывать часть строки, следующую после символа "?", например:

Полные адреса: 

/showtread.php?s=63843AB938CD944ED
/showtread.php?s=4058473C9498EF4949
/index.php?page=2
/index.php?page=1
/index.php?page=2&str=gu%3dru

Сокращенные адреса 

/showtread.php
/index.php

Отчет "Точки входа"

Точки входа, это страницы, с которых посетители Вашего сайта начинают с ним работу.

Кликнув на ссылку "[r]" Вы можете посмотреть страницы, ссылающиеся на заданную точку входа.

Отчет "Точки выхода"

Точки выхода, это страницы, на которых посетители Вашего сайта завершают работу с ним.

С помощью дополнительной опции "Исключать случайных" вы можете исключить посетителей просмотревших одну страницу. Часто, это посетители, случайно попавшие на ваш сайт, например с поисковой системы. Для этих посетителей "точка выхода" является "точкой входа"

Отчет "Доменные имена"

Отчет позволяет определить популярность доменных имен Вашего сайта, например:

По умолчанию, показываются только те доменные имена, по которым были зарегистрированы уникальные переходы (новый хост), но с помощью дополнительного меню, вы можете посмотреть полный список доменных имен. 

http://www.host.ru/
http://forum.host.ru/
http://host.ru/
http://new.host.ru/

Отчет "Первый уровень сайта"

Для большинства сайтов отчет позволяет определить наиболее популярные разделы, например:

По умолчанию, показываются только те разделы, по которым были зарегистрированы уникальные переходы (новый хост), но с помощью дополнительного меню, вы можете посмотреть полный список разделов. 

/
/cnstats/
/cncat/
/cnsearch_pro/
/download.xhtm

В адресах отбрасывается все, что идет после второго символа "/"

Отчет "Пути по сайту"

Данный отчет позволяет определить наиболее популярные последовательности просмотра страниц. Иногда это может быть одна страница.

Путь считается уникальным, если за указанный промежуток времени по нему прошел только один посетитель. В этом случае, вы можете получить дополнительную информацию о посетителе.

Отчет достаточно сложный и требует большого количества времени и памяти для расчетов.

Максимальное время расчета ограничивается, по умолчанию - 10 минут. Вы можете изменить это значение с помощью опции $STATS_CONF["slow_reports_time_limit"], например: 

$STATS_CONF["slow_reports_time_limit"]=1200;
// 20 минут (или 1200 секунд)

Максимальное количество используемой памяти зависит от настроек PHP. По умолчанию - 8Mb. Изменить это значение можно с помощью опции $STATS_CONF["slow_reports_memory_limit"], например: 

$STATS_CONF["slow_reports_memory_limit"]="64M"

Более подробнее об ограничении памяти вы можете прочитать на официальном сайте PHP.

Отчет "Глубина просмотра"

Отчет позволяет определить среднее количество страниц, которые просматривают ваши посетители.

Отчет "Ссылающиеся страницы"

В отчете показаны страницы, с которых посетители переходят на сайт. Фактически это страницы содержащие ссылки на Ваш сайт.

По умолчанию, показываются только те страницы, с которых были зарегистрированы уникальные заходы (новый хост), но с помощью дополнительного меню, Вы можете посмотреть полный список.

С помощью специальной формы, в нижней части отчета, Вы можете исключить страницы, мешающие просмотру отчета. Часто это страницы своего сайта или поисковые системы.

Пример списка исключаемых страниц: 

http://www.mysite.com
http://mysite.com
http://search.rambler
http://www.google
http://www.yandex
http://yandex
http://sm.aport.ru

Отчет "Ссылающиеся сервера"

Данный отчет очень похож на отчет "Ссылающиеся страницы", только в нем отображается не полный адрес страницы, а только адрес сервера, например: 

http://www.google.com/
http://www.google.com.tw/
http://www.yahoo.com/
http://www.dmoz.org/

Также как и в отчете "Ссылающиеся страницы", по умолчанию, показываются только те сайты, с которых были зарегистрированы уникальные заходы (новый хост), но с помощью дополнительного меню, Вы можете посмотреть полный список.

Отчеты "Переходы с поисковых систем", "Переходы с каталогов", "Переходы с рейтингов", "Переходы с популярных серверов"

Эта серия отчетов отображает список заранее заданных в системе сайтов, с которых достаточно часто осуществляются переходы. Сайты разбиты на три категории:

По умолчанию, показываются только те переходы, с которых были зарегистрированы уникальные заходы (новый хост), но с помощью дополнительного меню, Вы можете посмотреть полный список.

  1. Поисковые системы
  2. Интернет-каталоги
  3. Рейтинги сайтов

Четвертый отчет - это сумма трех описанных выше.

Отчет "История переходов"

Отчет позволяет проследить изменение количества переходов с популярных серверов за последние 150 дней.

Данный отчет требует предварительных расчетов. Если в лог-таблице есть данные для его построения, то вы увидите кнопку "Построить отчет". При клике по ней, будет произведены предварительные расчеты. Расчеты могут занять некоторое время.

При последующем построении, отчет не будет рассчитываться целиком, будут, рассчитаны только недостающие данные.

Для просмотра отчета вам необходимо выбрать сервера по которым хотите получить информацию и кликнуть на кнопку "Обновить"

Вы можете выбрать не более трех серверов.

Отчет "Поисковые фразы"

Отчет отображает список поисковых фраз, по котором находят страницы Вашего сайта в поисковых системах. Все поисковые фразы приведены к нижнему регистру.

По умолчанию, показываются только те фразы, по которым были зарегистрированы уникальные переходы (новый хост), но с помощью дополнительного меню, Вы можете посмотреть полный список фраз.

Отчет "Поисковые ссылки"

Отчет отображает список поисковых ссылок и фраз, по которым находят страницы Вашего сайта в поисковых системах. В отличие от предыдущего отчета регистр поисковых фраз не изменяется, у каждой фразы есть название поискового сервера и ссылка на страницу с результатом поиска.

По умолчанию, показываются только те ссылки, по которым были зарегистрированы уникальные переходы (новый хост), но с помощью дополнительного меню, вы можете посмотреть полный список ссылок.

Отчет "Найденные страницы"

Данный отчет отображает список страниц, на которые осуществлялись переходы с поисковых систем. Благодаря этому отчету Вы сможете оценить правильность написания текстов. Поймете, какие страницы сайта чаще находятся при работе посетителей с поисковыми системами.

По умолчанию, показываются только те страницы, по которым были зарегистрированы уникальные переходы (новый хост), но с помощью дополнительного меню, вы можете посмотреть полный список страниц.

В отчете отображаются следующие данные:

  • Поисковая система - поисковая система, с который был осуществлен один из переходов.
  • Найденные страницы - адрес страницы, на которую была осуществлены переходы с поисковых систем.
  • Поисковые фразы - фразы, по которой был сделан переход на эту страницу. Если на страницу были переходы более чем по одному слову, список слов отображается в свернутом списке, развернуть который можно, нажав на пиктограмму слева от надписи "Поисковые фразы".

С помощью пунктов "Полный URL" и "Сокращенный URL" можно отбрасывать часть строки, следующую после символа "?",

Внимание: В режимах "Сокращенный URL" и "Без учета доменов" построение отчета может занять продолжительное время и размер первой страницы отчета может быть значительным.

Отчет "Партнеры"

Данный отчет очень похож на серию отчетов "Переходы с популярных серверов", только в нем Вы можете оперативно, с помощью дополнительного меню, добавить Ваших партнеров.

Например, Вы обменялись ссылками с сайтов http://www.linksite.com/, тогда в список партнеров Вы добавляете две записи:

  • URL сайта - http://www.linksite.com/, Название: LinkSite
  • URL сайта - http://linksite.com/, Название: LinkSite

После того, как в отчете накопится более 5 партнеров, Вы сможете четко видеть, с кем Вам сотрудничать выгодно, а с кем не очень.

По умолчанию, показываются только те переходы, по которым были зарегистрированы уникальные переходы (новый хост), но с помощью дополнительного меню, Вы можете посмотреть полный список.

Отчет "IP адреса"

Отчет представляет список хостов, с которых был осуществлен доступ к Вашему сайту за выбранный период времени. Отчет отсортирован по количеству запросов. Отчет может использоваться для определения активности посетителей, выходящих с конкретного адреса или подсети. Так, например, Вы можете выяснить активность на Вашем сайте ваших сотрудников или сотрудников конкурента, правда, для этого необходимо знать соответствующие IP адреса.

В скобках указывается прокси-сервер. Кликнув по IP адресу, вы можете получить дополнительную информацию по IP адресу - страну (*), информацию whois, а также список страниц просмотренных с данного IP адреса. Таким образом, Вы можете отслеживать траекторию работы посетителя с Вашим сайтом, особенно удобно использовать формат "Сокращенный отчет".

(*) Страна определяется только, если, Вы загрузили базу данных стран. Подробнее в отчете "Страны".

Отчет "Сети"

С помощью данного отчета Вы можете определить количество посещений, сделанных из определенных сетей. Например, сети организации, города или даже страны.

Мы, например, используем этот отчет для определения популярность городского ресурса у посетителей различных Интернет-провайдеров города.

Список подсетей Вы можете задать с помощью пункта "Список подсетей" дополнительного меню. Количество добавляемых сетей ограничено длиной mysql запроса.

Отчет "Языки"

Отчет позволяет определить язык, установленный в системах Ваших посетителей. Язык определяется с помощью поля "Accept-Languages" HTTP запроса. См. отчет "Accept-Languages".

Отчет "Языки по странам"

Отчет "языки по странам" позволяет определить какие языки установлены в системах посетителей вашего сайта, в зависимости от страны. Отчет позволяет более точно описать аудиторию вашего сайта.

Отчет "Страны"

Отчет позволяет определить жители, каких стран, заходят на ваш сайт.

CNStats использует базу данных стран GeoIP (www.maxmind.com). Для правильного учета стран, вам необходимо проделать следующие шаги:

  • Скачайте базу данных (прямая ссылка) с официального сайта GeoIP
  • Распакуйте ее в каталог cnstats/geoip. Файл должен называться GeoIP.dat

Отчет строит интерактивную карту мира, выделяя страны в которых проживает основная часть ваших посетителей.

Отчет "Города"

Отчет позволяет определить жители, каких городов, заходят на ваш сайт.

CNStats использует базу данных GeoIPCity (www.maxmind.com). Для правильного учета стран, вам необходимо проделать следующие шаги:

  • Скачайте базу данных GeoIPCity (прямая ссылка) с официального сайта Alentum Software, Inc.
  • Распакуйте ее файл GeoIPCity.dat в каталог cnstats/geoip.

Отчет строит интерактивную карту мира, выделяя города, в которых проживает основная часть ваших посетителей. Размер точки на карте строится следующим образом:

Размер точки
(в пикселах)		Количество посещений
1от 1 до 5
3от 6 до 10
5от 11 до 100
7от 101 до 1000
9от 1001 до 10000
11более 10001

Отчет "Агенты"

Поле заголовка User-Agent содержит информацию о посетительском агенте, пославшем запрос. Данное поле используется для статистики, прослеживания ошибок протокола, и автоматического распознавания посетительских агентов. Хотя это не обязательно, посетительские агенты должны всегда включать это поле в свои запросы. Поле может содержать несколько строк, представляющих собой название программного продукта, необязательную косую черту с указанием версии продукта, а также другие программные продукты, составляющие важную часть посетительского агента. По соглашению, продукты указываются в списке в порядке убывания их значимости для идентификации приложения.

Например: 

Mozilla/4.0 (compatible; MSIE 5.5; Windows NT 4.0)
Mozilla/4.0 (compatible; MSIE 6.0; Windows NT 5.1)
             Opera 7.23 [en]
Mozilla/5.0 (Windows; U; Windows NT 5.0; fr; rv:1.6)
             Gecko/20040206 Firefox/0.8

Отчет "Accept-Languages"

Браузер, посредством поля Accept-Languages в специальном виде передает языковые метки.

Языковая метка идентифицирует естественный язык. Компьютерные языки в этот перечень не входят.

Синтаксис и регистр языковых меток HTTP тот же, что и определенный в RFC 1766. Языковая метка содержит одну или более частей: первичная языковая метка и последовательность субметок, которая может и отсутствовать: 

language-tag        = primary-tag *( "-" sub-tag )
primary-tag        = 1*8ALPHA
sub-tag
        = 1*8ALPHA

Пробел не допустим в метке, применение строчных и прописных букв не играет никакой роли. Перечень языковых меток контролируется IANA. Ниже приведены примеры языковых меток: 

en,
en-US,
ru,
ru;q=1.0,
en;q=0.9

где любые две буквы первичной метки представляют собой языковую аббревиатуру ISO 639 и две буквы исходной субметки соответствуют коду страны ISO 3166.

Отчет "Браузеры"

Отчет позволяет определить, какими браузерами и какими версиями браузеров пользуются посетители страницы.

По умолчанию, показываются только те браузеры, по которым были зарегистрированы уникальные переходы (новый хост), но с помощью дополнительного меню, Вы можете посмотреть полный список.

С помощью дополнительного меню Вы можете включить или выключить отображение версий браузеров.

Отчет "Роботы"

Отчет позволяет определить список роботов, которые запрашивали страницы вашего сайта. По большей части, это индексирующие роботы поисковых систем. С помощью дополнительного меню Вы можете изменять подробность данного отчета.

Для подробного отчета отображаются следующие данные:

  • Робот - имя поискового робота (например, Google Bot)
  • User-Agent - Идентификационная строка робота
  • Количество - количество страниц запрошенных роботом.
  • Последний визит - дата последнего визита робота
  • Первый визит - дата первого визита робота.

В сокращенной версии отчета отображается лишь список роботов. Кликнув по ссылке Вы, как и в подробной версии отчета перейдете к отчету "Просмотр лога" с заранее установленными фильтрами по полю User-Agent.

Отчет "Операционные системы"

Отчет позволяет определить, посетители каких операционных систем обращаются к страницам вашего сайта чаще всего.

По умолчанию, показываются только те операционные системы, по которым были зарегистрированы уникальные переходы (новый хост), но с помощью дополнительного меню, Вы можете посмотреть полный список.

Отчет "Сейчас на сайте"

Отчет отображает список посетителей, которые посещали страницы Вашего сайта за последние 4 минуты. С помощью дополнительного меню вы можете изменять подробность данного отчета.

Для подробного отчета отображаются следующие данные:

  • Дата - точная дата и время загрузки страницы
  • URL - адрес просмотренной страницы
  • Реферер - адрес ссылающейся страницы
  • IP - адрес посетителя
  • Прокси - Если посетитель использует, для подключения прокси-сервер, то отображается IP адрес прокси
  • User-Agent - Идентификационная строка браузера посетителя
  • Язык - Язык, установленный в системе посетителя
  • Страна - отображается только для посетителей просматривающих первую страницу (точку входа). Если же хост не уникален (это не первая страница, которую просматривает этот хост), то страна, для экономии ресурсов сервера, не определяется.

В сокращенной версии отображаются следующие данные:

  • Дата - точная дата и время загрузки страницы
  • IP - адрес посетителя
  • URL - адрес просмотренной страницы
  • Ссылающаяся страница - адрес ссылающейся страницы

Отчет "Размер базы данных"

Данный отчет является служебным и предоставляет информацию об изменении размера базы данных CNStats. Эта информация поможет вам правильно установить количество дней, за которые вы можете позволить себе хранить статистику.

Файлы данных

Для построения некоторых отчетов требуются определенные данные. Например, для построения отчета "Операционные системы" необходимо знать, как поле User-Agent зависит от операционный системы.

Например, Windows XP определяется по наличию строки "Windows NT 5.1", в Windows 2000 определяется по наличию строки "Windows NT 5.0"

Строки, по которым определяются операционные системы, называются "токенами".

Подобным образом строятся следующие отчеты:

  • Переходы с поисковых систем
  • Переходы с каталогов
  • Переходы с рейтингов
  • Переходы с популярных серверов
  • История переходов
  • Поисковые фразы
  • Поисковые ссылки
  • Найденные страницы
  • Броузеры
  • Роботы
